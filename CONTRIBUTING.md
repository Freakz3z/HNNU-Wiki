## 如何贡献

### 1. 创建新文件

#### 步骤一：确定文件位置

根据内容类型，将文件放在对应的目录中：

- **教程类内容** → `快速开始/`
- **基础知识** → `知识扫盲/`
- **技术专题** → `技术栈/`
- **校园信息** → `校园生活/`
- **竞赛经验** → `竞赛指南/`
- **工具介绍** → `工具箱/`

#### 步骤二：创建 Markdown 文件

在对应目录下创建 `.md` 文件：

```bash
# 示例：在快速开始目录下创建新文件
touch 快速开始/新教程.md
```

#### 步骤三：编写内容

文件头部应包含标题和简要描述：

```markdown
# 文章标题

## 简介

这里是文章的简要介绍...

## 正文

这里是正文内容...
```

### 2. 更新 SUMMARY.md

**SUMMARY.md 是 GitBook 的目录文件，必须更新它才能让新文件显示在目录中。**

#### SUMMARY.md 结构示例

```markdown
# Summary

## 快速开始

* [介绍](快速开始/介绍.md)
* [Python入门](快速开始/Python入门.md)
* [新教程](快速开始/新教程.md)  ← 新增的文件

## 知识扫盲

* [编程术语](知识扫盲/编程术语.md)
```

#### 添加新文件的步骤

1. **打开 SUMMARY.md**
2. **找到对应的章节**（在 `##` 标题下）
3. **添加新条目**，格式为：

```markdown
* [显示名称](相对路径/文件名.md)
```

**示例**：

```markdown
# 在"快速开始"章节下添加

* [VSCode 使用指南](快速开始/VSCode使用指南.md)
```

#### 重要提示

- ⚠️ 路径必须以 `/` 开头（绝对路径）
- ⚠️ 路径中不要有 `../` 这样的相对路径引用
- ✅ 正确：`* [教程](快速开始/教程.md)`
- ❌ 错误：`* [教程](./快速开始/教程.md)`
- ❌ 错误：`* [教程](../快速开始/教程.md)`

#### 创建子章节

如果需要创建子章节，使用嵌套列表：

```markdown
## 技术栈

* [前端](技术栈/前端/README.md)
    * [HTML基础](技术栈/前端/HTML基础.md)
    * [CSS基础](技术栈/前端/CSS基础.md)
* [后端](技术栈/后端/README.md)
```

### 3. 编辑现有文件

#### 修改错别字或补充内容

1. 找到需要修改的文件
2. 直接编辑文件内容
3. 保持原有的格式和风格
4. 提交修改

#### 添加章节

在现有文件中添加新章节时：

```markdown
# 原有标题

## 已有章节

## 新增章节

这里是新增的内容...
```

## 文件命名规范

### 基本规范

- 使用中文命名（适合中文文档）
- 使用有意义的名称
- 避免特殊字符
- 文件扩展名必须是 `.md`

### 示例

✅ 好的命名：
- `Python入门.md`
- `Git使用指南.md`
- `数据结构基础.md`

❌ 不好的命名：
- `new.md`（无意义）
- `test1.md`（无意义）
- `Python_入门.md`（避免下划线）

### 图片命名

图片应放在对应目录的 `image/` 或 `Image/` 文件夹中：

```
快速开始/
├── Python入门.md
└── image/
    ├── python-logo.png
    └── code-example.png
```

在 Markdown 中引用图片：

```markdown
![图片说明](image/python-logo.png)
```

## 内容规范

### Markdown 语法

#### 标题层级

```markdown
# 一级标题（文章标题，只有一个）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（小节）
```

#### 代码块

指定语言以获得语法高亮：

````markdown
```python
def hello():
    print("Hello, World!")
```

```javascript
console.log("Hello, World!");
```
````

#### 链接

内部链接（指向项目内的其他文件）：

```markdown
[显示文本](/路径/文件.md)
```

外部链接：

```markdown
[GitHub](https://github.com/Freakz3z/HNNU-Wiki)
```

#### 表格

```markdown
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 内容1 | 内容2 | 内容3 |
```

#### 列表

无序列表：
```markdown
- 项目1
- 项目2
  - 子项目2.1
  - 子项目2.2
```

有序列表：
```markdown
1. 第一步
2. 第二步
3. 第三步
```

### 内容质量要求

1. **准确性**：确保技术内容准确无误
2. **完整性**：提供完整的信息，不要遗漏关键步骤
3. **易懂性**：使用清晰的语言，适合初学者理解
4. **实用性**：内容应具有实际应用价值

## 提交规范

### Commit Message 格式

使用清晰简洁的提交信息：

```
<类型>: <简短描述>

<详细描述（可选）>
```

#### 类型（type）

- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 重构
- `test`: 测试相关
- `chore`: 构建/工具相关

#### 示例

```bash
# 添加新文件
git commit -m "docs: 添加 Python 异常处理章节"

# 修复错别字
git commit -m "fix: 修正 Git 入门中的错别字"

# 更新文档
git commit -m "docs: 更新 README.md 徽章"
```

### Pull Request 流程

详细的 PR 提交流程请参考：[如何提交 PR](/关于/手册/如何提交PR.md)

## 代码风格

### 缩进和格式

- 使用 2 个空格缩进（不要使用 Tab）
- 标题与内容之间空一行
- 章节之间空一行
- 代码块前后各空一行

### 示例

```markdown
# 标题

简介内容...

## 第一节

内容...

## 第二节

内容...

### 2.1 小节

内容...
```

### 中文排版

- 中英文之间添加空格：`学习 Python 编程`
- 中文与数字之间添加空格：`版本 2.0`
- 使用中文标点符号：`。、，；：` 等
